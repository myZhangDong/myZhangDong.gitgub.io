Created 2020-11-12| Updated 2020-11-24

uniapp 音视频 sdk api

1、创建会议
uni.emedia.mgr.createConference(11, ‘123456’, rec, recMerge)

2、加入会议
uni.emedia.mgr.joinConference(conferenceId, password)

3、给服务器发信令消息
uni.emedia.mgr.postMessage(confrId, data.detail.data, this.$refs.livePusher)

4、发布流
uni.emedia.mgr.publish(confrId, me.$refs.livePusher)

5、订阅流
uni.emedia.mgr.subscribe(confrId, stream, me.$refs.livePlayer)

6、退出会议
uni.emedia.mgr.exitConference(this.confrId)

其他api和小程序相同, joinRoom先没做支持

流程说明

1、 createConference 创建会议

2、 joinConference 加入会议

3、 publish 发布流，参数需要传入 livePusher 插件，sdk 调用插件 start 方法

4、 监听插件的onData 用户监听插件返回数据

5、 postMessage 交换信令，用户需要把插件回调出来的信令消息通过此api 发送给服务器

6、服务器返回的信令消息 sdk 内部通过onSignallingData 回调，直接调用插件的 setData 方法传给插件，不对外暴露

7、协商成功，显示画面

8、监听 onStreamAdded

9、subscribe 订阅流，需要传入 stream、livePlayer插件， sdk调用插件play方法

10、 重复 onData、postMessage 过程，sdk内部收到 onSignallingData，通过 rtcId 找到对应的插件 setData 与之进行交换信令

11、 协商成功显示其他人画面

uniapp集成多人通话

使用 uniapp 集成多人音视频SDK，生成android、ios客户端，需要下载环信uniapp原生插件emlivePusher、emlivePlayer。音视频SDK依赖IM SDK, 所以要先集成IM，并把IM SDK实例挂载在全局变量下：uni.WebIM = websdk, 可以参考uniapp demo utils/WebIM 文件。

导入插件

在插件市场下载好插件后，放到uniapp工程的nativeplugins目录下，如果没有此目录需要自己创建。导入好插件后需要在manifest.json/原生插件配置/本地插件中选择刚导入的插件，最后需要在运行/运行到手机或模拟器/制作自定义调试基座，这样插件就导入成功了，下次运行时在运行/运行到手机或模拟器/运行基座选择中选择自定义调试基座。

插件说明 （使用wiki中的文档）

插件属性

属性名称 类型 描述描述
ref String 页面根据ref找到该组件
style String 组件样式，参考原生组件支持样式
muted BOOL 是否静音
enableCamera BOOL 是否打开摄像头
devicePosition String 默认使用摄像头，front为前置，back为后置
videoWidth Number 视频分辨率宽
videoHeight Number 视频分辨率高
minBitrate Number 最小视频码率
maxBitrate Number 最大视频码率
mirror BOOL 是否镜像，Android独有，iOS没有
objectFit String 填充方式，fit和fill
isClarityFirst BOOL 是否清晰度优先
rtcLogToConsole BOOL 是否将Rtc日志输出到控制台

插件事件

事件名称 事件
bindstatechange 状态变化事件，detail={code, info}，code为状态码，info为描述
bindnetstatus 网络状态通知，detail={code, info}，code为状态码，info为描述
callbackData 数据回调通知，需要将回调出来的数据传给SDK
callbackStats 通话质量统计，detail={data}，data为通话质量数据，包括视频分辨率、码率、丢包率等

集成

uniapp音视频SDK的集成和微信小程序音视频SDK的集成类似，主要分为以下几个步骤：
1、引入SDK和、插件
2、创建会议
3、自己加入会议并发出邀请
4、自己发布发布流
5、其他人收到邀请加入会议
6、其他人发布流
7、监听SDK onMemberJoined、onStreamAdded事件，收到其他人的流之后订阅流
8、监听插件 callbackData、bindstatechange事件，将数据传给SDK
9、显示画面
10、退出会议
