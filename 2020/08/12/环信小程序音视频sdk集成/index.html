集成环信小程序SDK
Created 2020-08-12| Updated 2020-09-07|环信文档
Post View:|Comments:

小程序音视频原理

小程序通过SDK获取推拉流地址， 利用小程序的live-pusher、 live-player组件进行推拉流。 后台服务会进行编解码，从而能够和其他各端互通。SDK和信令服务进行通信, 从而获取到视频中其他人的状态, 比如进出会议（房间）、开始推拉流、静音、关闭摄像头等，回调给客户端。

准备工作

小程序要使用音视频功能，需要满足小程序的一些规定：

暂只针对国内主体如下类目：社交、教育、医疗、金融、汽车、政府、IT、工具的小程序开放，需要先通过类目审核，再在小程序管理后台，「开发」-「接口设置」中自助开通该组件权限。

注意： 微信小程序规定只有企业版才能使用音视频功能，个人版不能使用。
上线时还需要在小程序管理后台配置SDK中需要使用的域名：

  • request 合法域名: https://mprtc.easemob.com
  • socket 合法域名: wss://mprtc.easemob.com

此外，音视频SDK还依赖IM SDK，需要先集成IM，并将 IM 实例挂载在全局变量 wx 下。

开始集成

引入SDK

  • github: https://github.com/easemob/webim-weixin-xcx
  • 官网： https://www.easemob.com/download/im
    SDK位置：src/emedia/emedia_for_miniProgram.js

创建会议（房间）

创建会议（房间）有两种方式，这两种方式也是因为“会议”， “房间”这两个概念和业务场景产生的。

  • 第一种：调用 joinConference 创建会议，在回调中可以得到confr对象，创建者默认为管理员。

  • 第二种：调用joinRoom加入房间，可以指定房间的名称，并且在房间不存在时会自动创建，另外可以在加入房间时指定角色。

加入会议

加入会议的两种方式：

  • 第一种：加入会议分为两步：申请ticket, 使用ticket加入会议。

    • 1、申请ticket
    • 2、使用ticket加入会议

  • 第二种：使用密码加入会议

播放其他人的推流

首先在页面创建好之后我们要注册监听事件 onStreamAdded, 当其他人推流之后在会议中的人就会回调此事件。通过回调拿到 streamId 后，调用emedia.mgr.subStream(streamId)获取拉流地址，把拉流地址赋给live-player进行播放。

2、通过 streamId 获取拉流地址

其他回调

1、有人加入会议，已经在会议中的人将会收到回调。

2、有人退出会议，已经在会议中的人将会收到回调。

3、有人推流，比如有人调用LivePusherContext.start()之后。

4、有人停止推流，比如有人调用LivePusherContext.stop()之后。

5、角色改变。

6、媒体流发生变化，比如有人关闭摄像头。

7、会议退出，自己主动退出，服务端主动关闭；

回顾总结：

整个流程：
发起方：创建会议 => 加入会议 => 获取推流地址 => 推流 => 监听到其他流 => 获取拉流地址 => 播放
被邀请方：收到邀请消息 => 加入会议 => 获取推流地址 => 推流 => 监听到其他流 => 获取拉流地址 => 播放
